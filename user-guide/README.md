# User Guide

## Overview

The User Guide provides task-oriented documentation for using TaskFlow. It explains how users interact with the application, complete common activities, and navigate core product features throughout the project lifecycle.

The documentation is organized by feature so users can quickly locate instructions relevant to their responsibilities. Each guide focuses on a specific area of the application while maintaining consistency with the overall TaskFlow workflow.

Whether you are accessing TaskFlow for the first time or returning to perform daily project activities, the User Guide serves as the primary reference for working with the application.

---

## Who Should Use This Guide

The User Guide is intended for project managers, developers, testers, product owners, and other team members who use TaskFlow to plan, manage, and track project work.

Administrative configuration, deployment, and development topics are documented separately and are not covered in this guide.

---

## Documentation Structure

The User Guide follows the typical workflow of working in TaskFlow, beginning with initial onboarding and progressing through project planning, execution, collaboration, and reporting.

| Guide           | Purpose                                                                 |
| --------------- | ----------------------------------------------------------------------- |
| Getting Started | Introduces new users to the application and its basic workflow.         |
| Workspace       | Explains how work is organized across teams and projects.               |
| Dashboard       | Describes how to monitor project activity and identify priorities.      |
| Projects        | Covers project creation and management.                                 |
| Backlog         | Explains how upcoming work is organized and prioritized.                |
| Sprint          | Describes sprint planning and execution.                                |
| Board           | Explains how work is tracked through the workflow.                      |
| Task            | Covers the lifecycle of standard work items.                            |
| Bug             | Explains how defects are reported and managed.                          |
| User Story      | Describes how requirements are captured and tracked.                    |
| Notifications   | Explains how project updates and collaboration events are communicated. |
| Reports         | Describes project reporting and progress monitoring.                    |

---

## Recommended Reading Order

If you are new to TaskFlow, begin with **Getting Started** before continuing with the remaining guides.

Users responsible for managing projects should continue with **Workspace**, **Projects**, **Backlog**, and **Sprint**.

Team members responsible for daily execution should become familiar with **Board**, **Task**, **Bug**, and **User Story** before exploring **Notifications** and **Reports**.

Although each guide can be read independently, following this sequence provides a gradual introduction to the application and reflects the way projects are typically managed in TaskFlow.

---

## Documentation Conventions

The User Guide follows a task-oriented documentation model. Each document explains the purpose of a feature, when it should be used, and the recommended workflow for completing common activities.

Feature-specific implementation details, administrative configuration, and developer information are documented separately to keep user documentation focused on day-to-day product usage.
