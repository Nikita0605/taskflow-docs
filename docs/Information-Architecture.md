# Information architecture

## Purpose

This document defines the information architecture for the TaskFlow documentation portal. It explains how documentation is organized, why each section exists, and how users navigate between documents.

The goal is to help users locate information quickly while maintaining consistency across the documentation.

---

## Documentation goals

The TaskFlow documentation is designed to:

- Help new customers become productive quickly.
- Support daily product usage.
- Reduce support requests.
- Provide accurate technical reference information.
- Maintain consistent terminology.
- Scale as new product features are introduced.

---

## Audience analysis

TaskFlow documentation serves multiple audiences.

| Audience | Primary Goal | Documentation |
|-----------|--------------|---------------|
| New Customer | Learn the product | Product Overview, Getting Started |
| End User | Complete tasks | User Guide |
| Administrator | Configure the platform | Administrator Guide |
| Developer | Integrate with TaskFlow | API Reference |
| Support Engineer | Troubleshoot issues | Knowledge Base |

---

## Documentation principles

The information architecture follows these principles.

### User-first organization

Content is organized by user goals rather than by software modules.

### Progressive disclosure

Users begin with high-level concepts before moving to advanced topics.

### Single source of truth

Information is maintained in one primary location to prevent duplication.

### Consistent terminology

Every product term is defined in the glossary and used consistently throughout the documentation.

### Cross-referencing

Related documents reference each other to reduce repeated content.

The documentation follows the Diátaxis framework.

| Type | Purpose | Example |
|------|----------|----------|
| Tutorial | Learning by doing | Create your first project |
| How-to Guide | Solve a problem | Configure Slack integration |
| Reference | Technical information | Projects API |
| Explanation | Build understanding | Product Overview |

## Navigation strategy

Users are guided through the documentation based on their goals.

```
Overview
      ↓
Getting Started
      ↓
User Guide
      ↓
Administrator Guide
      ↓
Developer Guide
      ↓
API Reference
```

Each document should identify the recommended next step.
