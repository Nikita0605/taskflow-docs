# 1. About This Guide

The Quick Start Guide helps you create your first TaskFlow project with the minimum setup required.

In less than 10 minutes, you'll create a workspace, invite your team, build a project, plan a sprint, and add your first work items. By the end of this guide, you'll have a working environment that your team can immediately begin using.

If you need detailed explanations of TaskFlow features, continue with the User Guide after completing this guide.

---

# 2. Before You Begin

Before starting, make sure you have:

- A TaskFlow account
- A supported web browser
- A stable internet connection
- Permission to create a workspace

If your organization already uses TaskFlow, ask your Workspace Administrator to invite you instead of creating a new workspace.

---

# 3. Create Your Workspace

Your workspace is the home for your organization's projects, teams, documentation, dashboards, and settings.

To create a workspace:

1. Sign in to TaskFlow.
2. Select **Create Workspace**.
3. Enter a workspace name.
4. Choose your time zone.
5. Select **Create**.

Once the workspace is ready, you can begin inviting team members.

---

# 4. Invite Your Team

Projects are easier to manage when everyone joins from the beginning.

Open **Workspace Settings**, select **Members**, and choose **Invite Member**. Enter each team member's email address, assign an appropriate role, and send the invitation.

Each invited member receives an email with instructions for joining the workspace.

---

# 5. Create a Project

Projects organize work around a product, application, or initiative.

Create your first project by selecting **New Project**, entering a project name, choosing a project template, and selecting **Create Project**.

If you're new to TaskFlow, the **Scrum Project** template is a good starting point because it includes the boards and workflows required for Agile development.

---

# 6. Plan Your First Sprint

If your team follows Scrum, create a sprint before adding work.

Open your project, navigate to **Sprint Planning**, and select **Create Sprint**. Enter the sprint name, choose the start and end dates, and save your changes.

Your sprint is now ready for planning.

---

# 7. Add Work to the Backlog

The backlog contains the work your team plans to complete.

Start by creating a User Story.

For example:

> **As a customer, I want to reset my password so that I can regain access to my account.**

Break the User Story into smaller tasks such as designing the interface, developing the API, writing tests, and updating documentation.

Assign each task to the appropriate team member before starting the sprint.

---

# 8. Start Working

When sprint planning is complete, start the sprint and begin updating work as progress is made.

Move tasks through the workflow, update their status regularly, and record any defects as Bugs. Keeping work items current helps the entire team understand project progress without additional meetings or status updates.

---

# 9. Monitor Progress

As work progresses, TaskFlow automatically updates your project boards, dashboards, and reports.

Use the Dashboard to monitor sprint progress, review team workload, track open bugs, and identify blockers that require attention.

Checking the Dashboard regularly helps teams respond quickly to changing project conditions.
