# 1. About This Guide

## Overview

Welcome to TaskFlow.

This guide helps you set up your TaskFlow workspace and start managing software projects. It covers the initial setup process, introduces the core building blocks of the platform, and walks you through creating your first project.

The guide is written for new users. You don't need previous experience with TaskFlow to follow it.

By the time you finish, you'll have a working workspace, an active project, a planned sprint, and enough knowledge to begin collaborating with your team.

For advanced configuration, security settings, integrations, and system administration, refer to the **Administrator Guide**.

---

## What You'll Learn

This guide walks you through the essential setup process.

You'll learn how to:

- Create a workspace
- Invite team members
- Create your first project
- Select a project template
- Plan a sprint
- Add user stories and tasks
- Track project progress
- Use the project dashboard

Each section builds on the previous one, so it's best to follow the guide in order.

---

## Who Should Read This Guide

This guide is intended for anyone using TaskFlow for the first time.

Typical readers include:

- Project Managers
- Scrum Masters
- Product Owners
- Software Developers
- QA Engineers
- Technical Writers
- Engineering Managers

Workspace Administrators responsible for the initial platform setup should also complete this guide before moving to the Administrator Guide.

# 2. Before You Begin

Before creating your first workspace, make sure you have everything required to complete the setup.

### Requirements

You need:

- A supported web browser
- A stable internet connection
- A valid email address
- Permission to create a workspace (if you're joining an existing organization)

If your organization already uses TaskFlow, ask your Workspace Administrator to send you an invitation before creating a new account.

---

### Basic Knowledge

You don't need to be an Agile expert, but understanding a few common terms will make the setup process easier.

TaskFlow is built around concepts such as:

- Workspace
- Project
- Sprint
- Backlog
- User Story
- Task

These concepts are introduced throughout this guide, so don't worry if they're new to you.

---

# 3. System Requirements

TaskFlow is a web-based application and doesn't require any software installation.

### Supported Browsers

TaskFlow supports the latest versions of:

- Google Chrome
- Microsoft Edge
- Mozilla Firefox
- Safari

For the best experience, use the most recent version of your preferred browser.

---

### Screen Resolution

A minimum screen resolution of **1366 × 768** is recommended.

Larger displays provide more space for dashboards, boards, and project views.

---

### Internet Connection

TaskFlow requires an active internet connection.

A broadband connection is recommended, especially when working with large projects, dashboards, or file attachments.

---

# 4. Create Your TaskFlow Account

To use TaskFlow, you first need a user account.

1. Open the TaskFlow sign-up page.
2. Enter your name, email address, and password.
3. Accept the Terms of Service and Privacy Policy.
4. Select **Create Account**.
5. Verify your email address using the verification link.

After verification, your account is ready to use.

---

# 5. Sign In

1. Open the TaskFlow sign-in page.
2. Enter your email address and password.
3. Select **Sign In**.

After signing in, you'll see the TaskFlow Home page.

If this is your first login, you'll be prompted to create a workspace.

---

# 6. Create Your First Workspace

A workspace represents your organization inside TaskFlow.

It contains your projects, teams, members, documentation, dashboards, and settings.

To create a workspace:

1. Select **Create Workspace**.
2. Enter a workspace name.
3. Choose your organization type.
4. Select your preferred time zone.
5. Select **Create**.

Once the workspace is created, you can begin inviting team members.

---

# 7. Invite Your Team

Projects are easier to manage when the right people have access from the beginning.

To invite members:

1. Open **Workspace Settings**.
2. Select **Members**.
3. Select **Invite Member**.
4. Enter one or more email addresses.
5. Assign a role.
6. Send the invitation.

Invited members receive an email with instructions for joining the workspace.

---

# 8. Create Your First Project

Projects organize work around a product, application, or initiative.

To create a project:

1. Open your workspace.
2. Select **New Project**.
3. Enter a project name.
4. Add an optional description.
5. Choose a project template.
6. Select **Create Project**.

Your project is now ready for planning.

---

# 9. Choose a Project Template

TaskFlow provides templates that help teams get started quickly.

Available templates may include:

- Scrum Project
- Kanban Project
- Bug Tracking
- DevOps
- Blank Project

Choose the template that best matches the way your team works.

Templates can be customized later if your process changes.

---

# 10. Create Your First Sprint

If you're using Scrum, the next step is creating a sprint.

To create a sprint:

1. Open the project.
2. Go to **Sprint Planning**.
3. Select **Create Sprint**.
4. Enter the sprint name.
5. Set the start and end dates.
6. Select **Create**.

Your sprint is now ready to receive work items.

---

# 11. Add Your First User Story

User Stories describe features from the user's perspective.

For example:

> As a customer, I want to reset my password so that I can regain access to my account.

To add a User Story:

1. Open the backlog.
2. Select **New User Story**.
3. Enter the title and description.
4. Set the priority.
5. Save the story.

---

# 12. Create Tasks

Break each User Story into smaller tasks that can be assigned to team members.

Examples include:

- Design the user interface
- Develop the API
- Write unit tests
- Update documentation
- Verify the feature

Assign each task to the appropriate team member and update its status as work progresses.

---

# 13. Track Progress

As work moves through the sprint, TaskFlow automatically updates boards, dashboards, and reports.

Use these views to:

- Monitor sprint progress
- Track completed work
- Identify blocked tasks
- Review open bugs
- Monitor team workload

Keeping work items up to date ensures that project information remains accurate.

---

# 14. Explore the Dashboard

The Dashboard gives you a real-time view of project activity.

Depending on your role, you may see:

- Sprint progress
- Team workload
- Open bugs
- Pipeline status
- Recent deployments
- Upcoming releases

Dashboards can be customized to display the information most relevant to your work.
